Vocabulaires Core de l'UE SEMIC

Maintenus par la Commission européenne dans le cadre du programme ISA²/SEMIC. Légers, extensibles, alignés avec schema.org et Dublin Core.

Vocabulaire Domaine Namespace Préfixe Origine
Core Person Vocabulary Personnes physiques http://www.w3.org/ns/person# person: W3C EU
Core Business Vocabulary Organisations, personnes morales http://www.w3.org/ns/legal# legal: W3C EU
Core Location Vocabulary Adresses, géométries http://www.w3.org/ns/locn# locn: W3C EU
CPSV-AP Services publics, démarches http://purl.org/vocab/cpsv# cpsv: EU
CCCEV Critères, pièces justificatives http://data.europa.eu/m8g/ cv: EU

Exemples RDF commentés

Utilisation du Core Person Vocabulary + schema.org

# Déclaration des préfixes @prefix person: <http://www.w3.org/ns/person#> . @prefix schema: <http://schema.org/> . @prefix xsd: <http://www.w3.org/2001/XMLSchema#> . <https://data.example.gouv.fr/personne/p-001> a person:Person ; # Identité schema:givenName "Marie" ; schema:familyName "Dupont" ; person:birthName "Marie Martin" ; schema:birthDate "1985-04-12"^^xsd:date ; # Nationalité (référentiel Publications Office EU) person:citizenship <http://publications.europa.eu/resource/authority/country/FRA> ; # Lieu de naissance (lié à Core Location) person:placeOfBirth <https://data.example.gouv.fr/commune/75056> .
Bonne pratique : utilisez des URI pérennes sous votre domaine officiel data.monministere.gouv.fr. Les valeurs de listes contrôlées (pays, langues) doivent pointer vers les référentiels du Publications Office.

Utilisation du Core Business Vocabulary + W3C Org Ontology

@prefix legal: <http://www.w3.org/ns/legal#> . @prefix org: <http://www.w3.org/ns/org#> . @prefix schema: <http://schema.org/> . @prefix locn: <http://www.w3.org/ns/locn#> . <https://data.example.gouv.fr/org/dinum> a legal:LegalEntity, org:Organization ; legal:legalName "Direction interministérielle du numérique"@fr ; legal:alternativeName "DINUM"@fr ; legal:legalIdentifier "13001518700013" ; # SIRET legal:companyType <http://publications.europa.eu/resource/authority/legal-entity-type/GOV> ; schema:url <https://www.numerique.gouv.fr> ; legal:registeredAddress <https://data.example.gouv.fr/adresse/dinum> ; # Tutelle org:subOrganizationOf <https://data.example.gouv.fr/org/premier-ministre> .

Utilisation de CPSV-AP — démarche administrative

@prefix cpsv: <http://purl.org/vocab/cpsv#> . @prefix cv: <http://data.europa.eu/m8g/> . @prefix dct: <http://purl.org/dc/terms/> . @prefix xsd: <http://www.w3.org/2001/XMLSchema#> . <https://data.example.gouv.fr/service/carte-grise> a cpsv:PublicService ; dct:title "Immatriculation d'un véhicule"@fr ; dct:description "Obtenir un certificat d'immatriculation."@fr ; dct:language <http://publications.europa.eu/resource/authority/language/FRA> ; # Organisme compétent cv:hasCompetentAuthority <https://data.example.gouv.fr/org/ants> ; # Canal d'accès cpsv:hasChannel <https://data.example.gouv.fr/canal/ants-web> ; # Pièce requise cpsv:hasInput <https://data.example.gouv.fr/evidence/certificat-cession> ; # Résultat cpsv:produces <https://data.example.gouv.fr/output/certificat-immat> ; # Délai de traitement cv:processingTime "P3D"^^xsd:duration .

Le même organisme, en JSON-LD — compatible avec toute API REST

{ "@context": { "legal": "http://www.w3.org/ns/legal#", "schema": "http://schema.org/", "locn": "http://www.w3.org/ns/locn#", "legalName": "legal:legalName", "identifier": "schema:identifier", "url": "schema:url" }, "@type": "legal:LegalEntity", "@id": "https://data.example.gouv.fr/org/dinum", "legalName": "Direction interministérielle du numérique", "identifier": "13001518700013", "url": "https://www.numerique.gouv.fr" }
En-tête HTTP requis : Content-Type: application/ld+json — sans cela, le contenu ne sera pas reconnu comme du RDF par les parseurs.
